API Reference

Vendor API Documentation

These endpoints allow secure access to patient and FHIR data for approved vendors. All requests must include the apiKey header.

Authentication

All API requests must include the following HTTP header:

apiKey: YOUR_API_KEY_HERE

1. Lookup Patient

GET /api/lookup-patient

Returns a list of all matched patients based on provided criteria.

Query Parameters (required):

  • firstName (string)
  • lastName (string)
  • dob (YYYY-MM-DD)

Example Request:

curl -X GET "https://yourdomain.com/api/lookup-patient?firstName=John&lastName=Doe&dob=1980-01-01" \
  -H "apiKey: YOUR_API_KEY_HERE"

Example Response:

{
  "patient": [
    {
      "patient_first_name": "JOHN",
      "patient_last_name": "DOE",
      "zip":"12345",
      ...
    },
    {
      "patient_first_name": "JOHN",
      "patient_last_name": "DOE",
      "zip":"12346",
      ...
    }
  ]
}

2. Generate MRN (Create Lead)

POST /api/generate-mrn

Creates a new patient lead and returns the generated MRN and timestamps.

Required JSON Body:

{
  "firstName": "John",
  "lastName": "Doe",
  "dob": "1980-01-01",
  "zip": "12345",
  "phone": "555-1234",
  "email": "john.doe@example.com",
  "gender": "male",
  "city": "New York",
  "state": "NY",
  "address": "123 Main St"
}
{gender : male-or-female }

Example Request:

curl -X POST https://yourdomain.com/api/generate-mrn \
  -H "Content-Type: application/json" \
  -H "apiKey: YOUR_API_KEY_HERE" \
  -d '{
    "firstName": "John",
    "lastName": "Doe",
    "dob": "1980-01-01",
    "zip": "12345",
    "phone": "555-1234",
    "email": "john.doe@example.com",
    "gender": "male",
    "city": "New York",
    "state": "NY",
    "address": "123 Main St"
  }'

Example Response:

{
  "mrn": "123456789",
  "created_at": "2024-06-01T12:34:56Z",
  "updated_at": "2024-06-01T12:34:56Z",
  "epoch": 1712345678
}

3a. Request FHIR Data

POST /api/request-fhir-data?mrn=GENERATEDMRN

Triggers a FHIR data pull for a patient using their MRN. The MRN must be provided as a query parameter.

Query Parameter (required):

  • mrn (string) — The MRN (patient account number) to request FHIR data for.

Example Request:

curl -X POST "https://yourdomain.com/api/request-fhir-data?mrn=123456789" \
  -H "apiKey: YOUR_API_KEY_HERE"

Example Response:

{
  "status": "pending",
  "message": "FHIR data pull initiated"
}

3b. Request FHIR Data with Provider Info

POST /api/request-fhir-data-2

Submits a request to generate or retrieve FHIR data based on patient and provider identifiers sent in the request body.

Headers (required):

  • Content-Type (string) — Must be set to application/json.
  • apiKey (string) — Your unique API key for authentication.

Request Body (JSON, required):

The following JSON object must be sent in the request body:

{
  "mrn": "string",
  "prov_npi": "string",
  "prov_first_name": "string",
  "prov_last_name": "string"
}

Example Request (cURL):

curl -X POST \
  -H "Content-Type: application/json" \
  -H "apiKey: YOUR_API_KEY_HERE" \
  -d '{
    "mrn": "patient-mrn-12345",
    "prov_npi": "1234567890",
    "prov_first_name": "John",
    "prov_last_name": "Doe"
  }' \
  "https://yourdomain.com/api/request-fhir-data-2

Example Response (Hypothetical):

A successful request will likely return a status indicating the job has been queued. 

{"patient": {"mrn": {"root": "encrypted message", "extension": "extension val"}........ }

4a. PDF Status

GET /api/pdf-status?mrn=GENERATEDMRN

Checks the PDF file sync status for a given MRN. Returns whether the PDF is synced, pending, or not found.

Query Parameter (required):

  • mrn (string) — The MRN (patient account number) to check status for.

Example Request:

curl -X GET "https://yourdomain.com/api/pdf-status?mrn=123456789" \
  -H "apiKey: YOUR_API_KEY_HERE"

Example Responses:

Synced:

{
  "status": "synced",
  "info": {
    "files": [ ... ]
  }
}

Pending:

{
  "status": "pending",
  "info": {
    "message": "Please wait 15 minutes for the sync.",
    "files": {"fcount": 2}
  }
}

No Files Found:

{
  "status": "no-files-found"
}

Failed to Search:

{
  "status": "failed-to-search"
}

4b. Get FHIR JSON

GET /api/get-fhir-json

Retrieves a specific FHIR JSON Bundle from a Google Cloud Storage (GCS) location.

Query Parameter (required):

  • gcs_uri (string) — The full GCS path to the FHIR JSON file you want to retrieve.

Headers (required):

  • Content-Type (string) — Must be set to application/json.
  • apiKey (string) — Your unique API key for authentication.

Example Request (cURL):

curl -X GET ""https://yourdomain.com/api/get-fhir-json?gcs_uri=gs://connective_health_prod_feb7/2130735857111368_..._PATIENT_EVERYTHING.json" \
-H "Content-Type: application/json" \
-H "apiKey: YOUR_API_KEY_HERE"

Example Success Response (200 OK):

{
  "resourceType": "Bundle",
  "id": "4d20fe8f-dd31-43c1-8d86-08f4dd48976f",
  "type": "collection",
  "entry": [
    {
      "fullUrl": "urn:uuid:patient-79ff3b3951012c589b839fb5d08cbbff",
      "resource": {
        "resourceType": "Patient",
        "id": "patient-79ff3b3951012c589b839fb5d08cbbff",
        "meta": {
          "source": "https://connectivehealth.io",
          "profile": [
            "http://hl7.org/fhir/us/core..."
          ]
        }
        // ... remainder of Patient resource
      }
    }
    // ... other entries in the bundle
  ]
}

5. Generate Temporary Link

POST /api/get-temporary-link

Generates a time-limited access link for a vendor to retrieve a specific resource stored in Google Cloud Storage.

Note: here we can use the parsed response from pdf-status endpoint

Request Headers:

  • Content-Type: application/json (required)
  • apiKey: your-key (required)

Body Parameters (JSON):

Parameter Type Description Required
uri string The Google Cloud Storage URI (gs:// path) of the file to generate a link for Yes
expiration_time integer Duration in seconds for which the generated link should remain valid Yes

Example Request:

curl -X POST \
  -H "Content-Type: application/json" \
  -H "apiKey: your-key" \
  -d '{
    "uri": "gs://connective_health_prod_feb7/1753506022_717f6a6f-1f54-49e5-9f39-712b87ef3ecf_2025_07_26_PATIENT_SUMMARY.pdf",
    "expiration_time": 120
  }' \
  https://fhir-demo-564097302460.us-central1.run.app/api/get-temporary-link

Success Response (200 OK):

{
  "temporary_url": "https://storage.googleapis.com/...?GoogleAccessId=...&Expires=...&Signature=...",
  "expiration_time": 120,
  "expires_at": "2025-07-26T15:30:45Z"
}

Error Responses:

  • 400 Bad Request - Invalid request parameters
  • 401 Unauthorized - Invalid or missing API key
  • 404 Not Found - The specified file does not exist
  • 500 Internal Server Error - Server error while processing the request

Error Handling

  • All errors are returned as JSON with an error field and appropriate HTTP status code.
  • Example: 
    {
  "error": "Missing required fields: firstName, lastName, dob"
}
